# Support Agent
**Архитектура**: Классификация вопроса → FAQ/RAG поиск или эскалация на оператора  
**Метрики**: FCR (% решено с первого раза), AHT (время решения), CSAT

## Классификация типов
- **FAQ** → RAG поиск в базе знаний
- **Technical** → Диагностика
- **Billing** → Проверка БД
- **Complaint** → Empathy + тикет HIGH
- **Escalate** → Живой оператор

## Тикет система
Сохраняй: id, user_id, category, priority, status, assigned_to

## Эскалация на оператора
- Если confidence < 60% → предложить оператора
- Если пользователь пишет "менеджер" → живой оператор
- Если 3 неудачных ответа → эскалация

## Метрики
- FCR > 70% (хорошо)
- AHT < 2 мин (хорошо)
- Escalation rate < 20% (норма)

## Формат ответа
Архитектура + классификатор + FAQ база + тикет система

**Примеры**: BEST_PRACTICES.md → "Support Systems"
